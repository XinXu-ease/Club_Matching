# 📱 三种身份的导航栏 & 界面差异详解

## 一级导航栏对比

### 🎯 游客（未登录）

**导航栏显示：** 
```
┌──────────────────────────┐
│ 👈 返回首页              │
└──────────────────────────┘
```

**特点：**
- ❌ 无底部导航栏（mobile 无四栏导航）
- ❌ 只能看 Explore，无法进入其他功能区
- ⚠️ 任何操作都会弹窗提示"请先登录"
- 🔄 可从 Explore 的 banner 点击"逛逛"继续浏览

---

### 🎓 新生用户（已登录）

**导航栏显示：**
```
┌────────────────────────────────────────────┐
│ 🔍 Explore │ ⚡ Match │ 💬 Messages │ 👤 Profile │
└────────────────────────────────────────────┘
```

**页面访问权限：**

| 导航 | 页面 | 功能 |
|-----|------|------|
| 🔍 **Explore** | s-explore | 浏览所有社团、活动、招新 |
| | s-search | 搜索结果 |
| | s-filter | 筛选结果 |
| | s-club | 社团主页（查看 + ✅收藏 + 📲提问 + 📝报名） |
| | s-activity | 活动详情（查看 + ✅收藏） |
| | s-recruit | 招新详情（查看 + 📝报名） |
| **⚡ Match** | s-match-guide | 智能匹配引导页 |
| | s-match-quiz | 4步问答（问题 1-4） |
| | s-match-result | 匹配结果与推荐 |
| | s-apply | 报名页（从 Match 结果报名） |
| | s-apply-success | 报名成功页 |
| **💬 Messages** | s-messages | 我的消息中心 |
| | s-msg-detail | 某个社团的问答详情 |
| **👤 Profile** | s-profile | 新生 Profile（Tab：收藏/报名/浏览/提问） |
| | s-settings | 设置 + 【申请管理员】入口 |
| | s-auth-apply | 认证申请表单 |
| | s-auth-status | 认证进度查看 |

**操作限制：**
- ❌ 无法访问管理功能（不显示 Manage 导航）
- ✅ 可以收藏、报名、提问、匹配
- ✅ 可以在 Profile 申请管理员身份

---

### 🏛️ 社团管理员（已认证）

**导航栏显示：**
```
┌────────────────────────────────────────────┐
│ 🔍 Explore │ 🏛️ Manage │ 💬 Messages │ 👤 Profile │
└────────────────────────────────────────────┘
     (与新生相同)     (替代 Match)      (与新生相同)    (不同)
```

**页面访问权限：**

| 导航 | 页面 | 功能 |
|-----|------|------|
| 🔍 **Explore** | s-explore | 浏览社团（依然可用，与新生相同） |
| | s-club | 社团主页 + 🔧【编辑】按钮（只显示给管理者） |
| **🏛️ Manage** | s-manage | 管理台首页（核心枢纽） |
| | s-edit-club | 编辑社团主页信息 |
| | s-publish | 发布招新公告 |
| | s-applications | 报名表管理 & 审核 |
| | s-faq | FAQ 和用户留言管理 |
| | s-stats | 数据统计面板 |
| **💬 Messages** | s-messages | 用户咨询消息（新生问的问题） |
| | s-msg-detail | 查看某一条咨询 + 【回复】 |
| **👤 Profile** | s-profile-admin | 社团管理员 Profile （不同UI） |
| | s-settings | 设置 + 【切换身份】选项 |

**特殊能力：**
- ✅ Match 导航被 Manage 替代（不再需要做匹配，因为要运营社团）
- ✅ 可以编辑自己的社团主页（编辑链接在 Explore 的 s-club 页面右上角）
- ✅ 可以发布招新公告（公告推送到 Explore，让用户看到）
- ✅ 可以查看报名表并审核
- ✅ 可以回复用户留言
- ✅ 可以看数据统计（访问量、转化率等）
- ✅ 可以下载报名表
- ✅ 如果管理多个社团，可在 Profile 切换身份

## 三、核心流程场景对比

### 场景 1：用户想收藏一个社团

#### 游客
```
Explore → 点击社团卡片 → s-club
→ 【点击收藏按钮】
→ ❌ 弹窗："请先登录"
→ 返回 s-welcome 登录
```

#### 新生
```
Explore → 点击社团卡片 → s-club
→ 【点击收藏按钮】
→ ✅ "已收藏 ❤"
→ Profile → 我的收藏 → 看到此社团
```

#### 管理员
```
Explore → 点击自己管理的社团 → s-club
→ 右上角 [✏️ 编辑] 按钮
→ s-edit-club （编辑主页）
或：【收藏】按钮与新生相同
```

---

### 场景 2：想报名加入招新

#### 游客
```
Explore → 【看招新banner】
→ s-recruit 招新详情
→ 【立即报名】
→ ❌ "请先登录"
→ 返回登录
```

#### 新生
```
Explore → 【看招新banner】
→ s-recruit 招新详情
→ 【立即报名】
→ s-apply 报名表单
→ 填写信息
→ 【提交】
→ s-apply-success ✅
→ 社团管理员在 s-applications 看到此报名
```

#### 管理员
```
s-manage 管理台
→ 【快捷卡：查看报名】
→ s-applications 报名表
→ 看到所有报名人，包括刚才提交的报名
→ 【标记审批状态】：通过/未录取
或：就充当新生去报名（如报名其他社团）
```

---

### 场景 3：想做社团智能匹配

#### 游客
```
Explore → 【banner："做匹配"】
→ ❌ "请先登录"
```

#### 新生
```
Explore → 【banner："做匹配"】
→ s-match-guide "找到更适合你的社团"
→ 【开始匹配】
→ s-match-quiz 4步问答
  ├─ 第1步：兴趣方向
  ├─ 第2步：参与目标
  ├─ 第3步：时间投入
  └─ 第4步：个人风格
→ 【提交】
→ s-match-result "匹配结果"
  ├─ 你的画像：低投入/表达型...
  ├─ 推荐社团 × 3-5个
  ├─ 【查看主页】→ s-club
  ├─ 【收藏】→ Profile
  └─ 【报名】→ s-apply
```

#### 管理员
```
❌ 无 Match 导航
（因为管理员已获得社团权限，不需要再做匹配）

但如果想了解用户数据：
→ s-manage 管理台
→ 📊 数据卡片 "被匹配次数"
→ 查看有多少用户通过匹配看到我的社团
```

## 六、认证流程（重要）

**新生 → 管理员的转变：**

```
1️⃣ 新生 在 Profile → Settings → 【申请社团管理员】
   ↓
2️⃣ 进入 s-auth-apply
   ├─ 填写：社团名、职位、身份证件、证书、电话
   ├─ 【预览】
   └─ 【提交申请】
   ↓
3️⃣ 进入 s-auth-status 查看进度
   ├─ 显示：✅ 申请已提交
   ├─ 显示：⏳ 审核中（3-7个工作日）
   └─ 显示：⏳ 审核完成（待通知）
   ↓
4️⃣ 🔔 审核通过后
   ├─ 发送通知给用户
   ├─ 用户回到 Profile → Settings
   ├─ 现在能看到：【切换身份】选项
   └─ 点击 → 身份从"新生"切换到"社团管理员"
   ↓
5️⃣ 切换后
   ├─ 底部导航从"| Explore | Match |"变成"| Explore | Manage |"
   ├─ Profile 变成 s-profile-admin
   ├─ 可以进入 s-manage 管理台
   └─ 完整的编辑/发布/数据功能全部解锁
```

---

## 总结

✅ **游客** = 只读浏览，所有写操作都被阻止  
✅ **新生** = 完整的用户功能（读+写），可申请升级管理员  
✅ **管理员** = Manage 权限解锁，Match 被替代为 Manage，Profile 变为管理员版本
